# リポジトリ概要

このリポジトリは、**Claude Codeを中心とした「マルチエージェントAI開発環境」の構築キット（テンプレート）**です。

Claude Codeをメインの指揮者（Orchestrator）とし、他の特化型AIツール（Codex CLI、Gemini CLI）やサブエージェントを連携させて、効率的に開発を行うための設定やワークフローがまとめられています。

## 主な役割と機能

### 1. AIエージェントの連携（オーケストレーション）

各AIツールがそれぞれの強みを活かして役割分担します。

*   **Claude Code**: プロジェクト全体の進行、ユーザーとの対話、タスクの調整を担当します。
*   **Codex CLI**: 設計、深い推論、デバッグなど、高度な技術的判断を担当します。
*   **Gemini CLI**: 最新情報のWebリサーチ、画像/PDFなどのマルチモーダル分析を担当します。
*   **Subagents**: コンテキスト（メモリ）を節約するため、特定のタスクを裏で並列処理します。

#### 連携の仕組み（3層のオーケストレーション）

この環境では、単なるツール呼び出しだけでなく、以下の3つのレベルでAI間の連携を実現しています。

##### 1. Hooks（提案型・リアクティブ）
ユーザーの入力に反応して、適切なツールの使用を提案する仕組みです。

*   **動作**: `.claude/hooks/agent-router.py` がユーザー入力を監視。
*   **トリガー**:
    *   「設計」「デバッグ」→ Codexの使用を提案
    *   「リサーチ」「最新」→ Geminiの使用を提案
*   **効果**: Claudeのコンテキストにヒント（実行コマンド例）を注入し、ツール使用を促します。

##### 2. Skills / Agent Teams（構造型・プロアクティブ）
特定のタスク（`/startproject` など）において、複数の専門エージェントをチームとして組織化し、能動的に連携させる仕組みです。

*   **並列実行**: 例えばプロジェクト開始時には、「Researcher（調査担当・Gemini）」と「Architect（設計担当・Codex）」という2つのエージェントを同時に起動します。
*   **双方向通信**: これら2つのエージェントは互いに通信可能です。
    *   Researcherが見つけた制約をArchitectに伝える
    *   Architectが設計に必要な追加調査をResearcherに依頼する
*   **役割分担**: Claude（Lead）はこれらを統括し、最終的な計画の統合と承認のみを行います。これにより、単独で処理するよりも深く、多角的な検討が可能になります。

##### 3. Subagent Pattern（委譲型・バックグラウンド）
時間のかかる処理や、大量のトークンを消費するタスクを裏方に任せる仕組みです。

*   **コンテキスト分離**: メインのClaudeの会話履歴（メモリ）を汚さずに、別の独立したコンテキストで作業を行います。
*   **用途**: 大規模なリファクタリング案の検討、広範囲のコード解析、詳細なデバッグ調査など。
*   **メリット**: メインのClaudeは軽量な状態を保ちつつ、重い処理結果だけを受け取ることができます。


### 2. 開発ワークフローの自動化（Skills）

以下のようなコマンド（スキル）を通じて、AI同士が協力してタスクをこなす手順が定義されています。

#### `/startproject` — プロジェクト開始
マルチエージェント協調でプロジェクトをセットアップします。
1. **Gemini**: リポジトリ分析・事前調査
2. **Claude**: 要件ヒアリング・計画作成
3. **Codex**: 計画レビュー・リスク分析
4. **Claude**: 具体的なタスクリスト作成

#### `/plan` — 実装計画作成
要件を具体的な実装ステップに分解します。
*   ファイル単位の変更内容
*   依存関係やリスクの洗い出し
*   完了を確認するための検証基準
これらを明確にした実装計画を出力します。

#### `/tdd` — テスト駆動開発
Red-Green-Refactorサイクルで堅牢なコードを実装します。
1. テストケース設計
2. 失敗するテスト作成（Red）
3. 最小限の実装（Green）
4. リファクタリング（Refactor）

#### `/checkpointing` — セッション永続化
作業の区切りでセッションの状態を保存します。
*   基本: 履歴ログの保存
*   完全: git履歴・ファイル変更を含む保存
*   分析: 再利用可能なスキルパターンの発見

#### `/codex-system` — Codex CLI連携（技術的相談）
設計判断・デバッグ・トレードオフ分析に使用します。
「どう設計すべき？」「なぜ動かない？」といった深い技術的な問いに対して、Codexの推論能力を活用します。

#### `/gemini-system` — Gemini CLI連携（リサーチ）
最新情報のWebリサーチや、画像/PDFなどのマルチモーダル処理に使用します。
「このライブラリの最新ドキュメントを調べて」「このPDFを要約して」といったタスクに適しています。

#### `/simplify` — コードリファクタリング
複雑になったコードを、機能やライブラリの制約を維持したまま簡潔化・可読性向上させます。

#### `/design-tracker` — 設計決定追跡
アーキテクチャや実装に関する重要な決定を自動的に記録し、`DESIGN.md` 等に反映させます。

### 3. プロジェクト構成の提供

`.claude`, `.codex`, `.gemini` といった設定ディレクトリを含んでおり、これらを自分のプロジェクトにコピーすることで、すぐにマルチエージェント開発環境を整えることができます。

## まとめ

「Claude Code単体で開発するのではなく、他のAIツールも適材適所で使い分け、チームのように協力させて開発を進めるためのセットアップ環境」を提供します。
